# 📄 PRD (Product Requirements Document)

## 📌 서비스명 (가칭)
**EduConnect (교육 운영 연결 앱)**

---

# 1. 🎯 목적 (Goal)

대규모 오프라인 교육(약 100명)에서  
운영진과 교육생 간 커뮤니케이션 효율을 높인다.

### 핵심 문제
- ❌ 출석 체크 수기 관리 → 비효율
- ❌ 질문이 운영진 개인에게 분산됨
- ❌ 실시간 대응 어려움

### 해결
- ✅ 사진 기반 출석 자동화
- ✅ 질문 중앙 집중 관리

---

# 2. 👥 사용자 정의

## 2.1 교육생 (User)
- 회원가입 후 출석 및 질문 기능 사용

## 2.2 운영진 (Admin)
- 출석 현황 관리
- 질문 응답 및 관리

---

# 3. 🧩 핵심 기능 (MVP Scope)

| 기능 | 교육생 | 관리자 |
|------|--------|--------|
| 회원가입 | O | - |
| 출석 체크 | O | 조회 |
| 질문 등록 | O | 조회/답변 |
| 질문 답변 확인 | O | - |
| 교육생 관리 | - | O |
| 출석 현황 관리 | - | O |

---

# 4. 📱 사용자 플로우

## 4.1 교육생
1. 회원가입
2. 로그인
3. 출석 체크 (사진 업로드)
4. 질문 등록
5. 답변 확인 (팝업)

## 4.2 관리자
1. 로그인
2. 교육생 리스트 확인
3. 출석 현황 확인
4. 질문 리스트 확인
5. 답변 작성

---

# 5. 🧱 상세 기능 정의

## 5.1 회원가입

### 입력 정보
- 이름 (텍스트)
- 분과 (1~10 선택)
- 로그인 방식:
  - 카카오
  - Google
  - 이메일

### 기능
- 가입 시 자동 로그인
- 관리자 페이지에 실시간 반영

---

## 5.2 출석 체크 기능

### 교육생 화면
- 안내 문구 표시  
  👉 "오늘 아침 출석미션을 대강당에서 확인하세요!"
- 사진 업로드 (1장)
- 제출 버튼

### 동작 로직
- 하루 1회 제출 가능
- 제출 시:
  - 날짜 자동 저장
  - 시간 자동 기록

### 관리자 화면

#### 📊 출석 현황 대시보드
구성:
1. 분과별 출석 완료 이미지 grid
2. 미출석자 리스트
3. 분과별 완료율 (%)
4. 전체 완료 순위 (게임 요소)

---

## 5.3 출석 데이터 구조
- user_id
- 날짜
- 업로드 이미지 URL
- 업로드 시간

---

## 5.4 질문(FAQ) 기능

### 교육생 화면

#### 입력 항목
- 카테고리 선택:
  - 근태
  - 교육운영
  - 숙소
  - 건강
  - 기타

- 질문 내용 (텍스트)

#### 기능
- 제출 후 본인 질문만 조회 가능
- 답변 달리면 팝업 표시

---

### 관리자 화면

#### 📋 질문 리스트
구성:
- 카테고리 필터
- 작성자 이름
- 작성 시간
- 질문 내용
- 답변 상태

### 관리자 액션
- 답변 작성
- 상태 변경:
  - 미답변
  - 답변 완료

### 알림 방식
- 교육생 로그인 시  
👉 "답변이 등록되었습니다" 팝업 노출

---

## 5.5 교육생 관리 기능

### 관리자 페이지

#### 📋 교육생 리스트
구성:
- 이름
- 분과
- 출석 상태
- 메모 필드

### 기능
- 메모 작성 가능

- 검색 기능
